入门指南

欢迎使用FTX工具套件。本指南将帮助您快速开始使用我们的工具。

系统要求（全部）

在开始之前，请确保您的系统满足以下要求：

// 最低系统要求
操作系统: Windows 7+ / macOS 10.14+ / Linux (Ubuntu 18.04+)
内存: 4 GB RAM (推荐 16 GB)
存储空间: 1 GB 可用空间
网络: 稳定的互联网连接
权限: 管理员/root权限（用于安装）/无视风险安装权限

许可证激活

要激活FTX工具，您需要有效的许可证代码。您可以通过以下方式获取：

  1. 从官方获取公开许可证
  2. 联系团队获取试用许可证
  3. 参与我们的开发者计划获取免费许可证
// 激活流程示例
1. 打开FTX工具启动器
2. 点击"许可证验证"按钮
3. 输入您的许可证代码
4. 点击"激活"按钮
5. 等待验证完成

安装与配置（适配中）

本节将指导您完成FTX工具的安装和初始配置过程。

Windows安装

对于Windows用户，请按照以下步骤安装：

# 1. 下载安装程序
Invoke-WebRequest -Uri "https://github.com/ProgrammerLiSichen/FTX_Tools" -OutFile "ftx-installer.exe"

# 2. 运行安装程序（需要管理员权限）
Start-Process -FilePath "ftx-installer.exe" -Verb RunAs

# 3. 按照安装向导完成安装
# 4. 启动FTX工具
Start-Process -FilePath "C:\Program Files\FTX Tools\ftx-launcher.exe"

配置文件（适配中）

安装完成后，您可以在以下位置找到配置文件：

// Windows 配置文件路径
%APPDATA%\FTX Tools\config.json

// macOS / Linux 配置文件路径
~/.config/ftx-tools/config.json

// 配置文件示例
{
  "license_key": "YOUR_LICENSE_KEY",
  "auto_update": true,
  "theme": "dark",
  "log_level": "info",
}

故障排除

常见问题及其解决方法。

常见问题

// 问题1: 许可证验证失败
原因: 许可证代码错误或过期
解决方案:
  1. 检查许可证代码是否正确
  2. 确认许可证是否在有效期内
  3. 联系支持团队获取帮助

// 问题2: 工具启动缓慢
原因: 系统资源不足或配置文件过大
解决方案:
  1. 关闭不必要的应用程序
  2. 清理临时文件
  3. 重置配置文件

// 问题3: 更新失败
原因: 网络问题或权限不足
解决方案:
  1. 检查网络连接
  2. 以管理员身份运行工具
  3. 手动下载更新包

日志文件

如果遇到问题，请检查日志文件以获取更多信息：

# Windows 日志路径
%APPDATA%\FTX Tools\logs\ftx.log

# macOS / Linux 日志路径
~/.local/share/ftx-tools/logs/ftx.log

# 查看最近日志
tail -f ~/.local/share/ftx-tools/logs/ftx.log

# 清除日志文件
ftx logs --clear

认证接口

用户登录和身份验证接口。使用此接口获取访问令牌，用于其他API调用。

请求

参数 类型 必填 描述
username string 是 用户名或邮箱地址
password string 是 用户密码
remember boolean 否 是否记住登录状态
// 请求示例
POST /auth/login
Content-Type: application/json

{
  "username": "user@example.com",
  "password": "your-password",
  "remember": true
}

响应

// 成功响应示例
HTTP/1.1 200 OK
Content-Type: application/json

{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 3600,
  "user": {
    "id": 123,
    "username": "user@example.com",
    "role": "admin"
  }
}
// 错误响应示例
HTTP/1.1 401 Unauthorized
Content-Type: application/json

{
  "success": false,
  "error": "Invalid credentials",
  "code": 401
}
